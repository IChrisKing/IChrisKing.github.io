一个简单的内外网连通实验

networkcjdns
  • Linux
  • cjdns
  • network
  • 网络

整体网络结构描述

两个网络

cjdns网络

这个网络中包括两台主机：

  1. Linux主机
  2. 树莓派A
    这两个主机依靠cjdns网络连接，树莓派A将Linux主机作为离岸点

内网

这个网络中包括两台主机：

  1. 树莓派A
  2. 树莓派B
    这两个主机依靠内网连接，他们在同一个内网网段中

三个主机

Linux主机

  1. 作为普通点，连接一个离岸点，获取到一个离岸点分配的ipv4地址

  2. 作为离岸点，给树莓派A分配ipv4地址192.168.254.2

树莓派A

  1. 作为cjdns网络中的普通点，将Linux主机设置为自己的离岸点，得到Linux主机分配给它的ipv4地址

  2. 作为与树莓派B在同一内网中的一台机器，有内网地址

树莓派B

作为内网内的机器，有内网地址

cjdns配置

Linux主机的配置

主要修改IpTunnel段的配置

outgoingConnections段是Linux主机作为cjdns网络中的普通点，申请离岸时需要的信息。Linux主机的离岸点是”publicKey”为”wfzyzrc0q4g83y0dgzxx1l862u0lscucj75yw9q1ymbltzwh2fq0.k“的这个点。
allowedConnections段是Linux主机作为离岸点，为树莓派A分配ipv4地址时需要的信息。树莓派A将被分配到ipv4地址192.168.254.2。
prefixes4和exceptions4共同构成树莓派A代理的内网网段信息。
prefixes4除去exceptions4后的网段就是树莓派A代理的内网网段。
此处没有exceptions4的信息，写在这里仅供格式参考。

树莓派A

主要修改IpTunnel端的配置

outgoingConnections段是树莓派作为cjdns网络中的普通点，申请离岸时需要的信息。树莓派A的离岸点就是Linux主机。

树莓派B

树莓派B不在cjdns网络中，不需要配置。

路由设置

Linux主机

需要在Linux主机上添加到169.254.0.0/16网络的路由信息，让前往这个网段的信息走cjdns网络（走tun0设备）。

列一下路由信息，看看设置效果

打开路由转发功能
/etc/sysctl.conf

树莓派B

设置树莓派A为默认路由

查看设置效果

实验

从Linux主机上ping树莓派B的内网地址169.254.106.46
数据包通过cjdns网络从Linux主机传递到树莓派A，然后通过路由转发，由连接cjdns的网卡传递到内网网卡，进而发送给树莓派B。

排错

如果ping不通，依次查看：

  1. pc的ip route,看看ping的地址是不是走了tun0
  2. 树莓派的转发功能是否打开了
  3. 树莓派B是否将树莓派A设置为默认路由
